System Architecture Overview

User Interface Layer

Claude Code CLI with slash commands (/morning, /update, /reflect)

Integration Layer

MCP servers connecting to external services (TickTick, Supabase)

Memory Layer

Omni-Cortex for session persistence and cross-project context

Data Layer

Supabase for historical storage, embeddings, and analytics

How It Works

The system creates a unified workflow where tasks flow from TickTick through daily commands, get tracked in memory, and persist to Supabase for long-term analysis.

Each layer serves a specific purpose: UI for interaction, MCPs for data access, memory for session continuity, and database for historical patterns.

The result is a system that learns your productivity patterns and improves recommendations over time.
